## Summary

Substantive PR that lands the planned `SqlDialectConfig` carrier refactor (Phase 1), adds the `MySqlBackslashEscapes` flag to attribute and carrier (Phase 2), branches LIKE emit on the flag (Phase 3, option 3.B), and adds default-mode-MySQL container regression coverage (Phase 4). The mechanical refactor across `SqlAssembler.cs` / `SqlExprRenderer.cs` is consistent and the LIKE-emit math (escape pass at parse time stays ANSI; doubling happens only at render time for MySQL+default) is correct end-to-end. Manifest drift is contained to the MySQL dialect (PG/SQLite drift in the master..HEAD diff comes from rebased master commits, not this branch). The most significant concrete findings are: (1) the back-compat `Render(..., SqlDialect)` overloads silently default `MySqlBackslashEscapes` to `true` so any non-test caller that constructed a `SqlDialect.MySQL` without going through the carrier will now silently emit doubled-form (no such site exists today, but the API is a footgun); (2) Phase 4 only exercises the `MySqlBackslashEscapes=true` path against the default-mode container â€” the planned opt-out test (false on default-mode with `SET sql_mode=...,NO_BACKSLASH_ESCAPES`) was not implemented; (3) two doc-comment updates the plan called out (`SqlExprNodes.LikeExpr.NeedsEscape`, `SqlLikeHelpers.EscapeLikeMetaChars`) were not made. None of the findings are blockers; all are tractable in this PR or as small follow-ups.
